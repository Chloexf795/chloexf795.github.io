WebLinguo

Teammates: David, Ian, Chloe, Emma [DICE]

What we want to make?

A plugin to help users learn a second language while browsing websites.

This prototype addresses user needs by integrating language learning into daily browsing. It enables contextual translation, vocabulary tracking, and pop-up quizzes to enhance user engagement.

Three Requirements

Remembering (History of Translated Words)
The plugin will store a history of words that users have previously translated. This helps users retain new vocabulary by providing an easy way to review past translations.

Translation (Instant Word Meaning)
Users can translate words from one language to another while browsing any website. This allows them to understand unfamiliar vocabulary in real time without disrupting their reading flow.

Quizzes (Active Learning Feature)
The plugin will quiz users on previously searched words using multiple-choice. This encourages active recall, reinforcing learning beyond passive translation.

How My Prototype Addresses the Requirements?

Remembering (History Feature)
As shown in the third video, when users click on the history button, a pop-up appears displaying a table of previously searched words. It includes translations, the number of times a word was searched, whether the word was used in a quiz, and whether the user got it correct. This structured approach helps learners track their progress and review words they might have struggled with.

Translation (Search and Definition Popup)
In the second video, the translation feature is visible at the top-right of the screen. User could search words there and the meaning and grammatical details will then appear. This makes it easy for users to get instant translations while keeping the website readable.

Quizzes (Randomized Pop-Ups)
The fifth video shows the quiz feature, which randomly pops up to test users on previously translated words. The quiz presents multiple-choice questions with four options, reinforcing learning through recall and recognition. This method helps users actively engage with vocabulary rather than just passively consuming translations.

Prototype Demonstration

By clicking on the switch, users can toggle WebLinguo on and off.

This video shows how the translation feature works. Users simply type the words they want to translate in the top-right search bar. If they want to translate a sentence instead of just words, they could click the translation button at the bottom-left, and the translation window will pop-up as shown.

The history feature allows users to review words they have translated before. This helps in vocabulary retention and language practice.

Once users click on the setting button, the setting window will pop up. Users can customize the plugin by their needs. For example, choose the language they use, set the frequency for quiz, and whether record the history by privacy.

The quiz feature helps reinforce learning through active recall. The quiz is multiple-choice, and the users could press the button to choose the answer. If the answer they chose is wrong, it would flip as "X" and then flip back. If the answer they chose is correct, the correct answer will flip for a second for memory strengthening, and then it will jump to next word. After the whole quiz is finished, the final page would display, showing the overall accuracy rate.

UX Research Process

We conducted one-on-one interviews (3 Fictional Inquiry and 1 Semi-Structured) with users who were actively learning a new language. These interviews focused on their browsing habits, how they currently translate words, and their pain points in digital language learning.

Team Analysis

Our team used axial coding to analyze user feedback. We first used sticky notes to categorize pain points, feature requests, and usability concerns. Then, we grouped user insights based on major themes such as remembering translated words, instant translations, and interactive quizzes. This helped us prioritize features that users found most beneficial and refine our prototype accordingly.

Results from UX Evaluation

What Worked Well:

  • The exit button ("X") was intuitive and clear; users immediately recognized how to exit each page.
  • Icons for settings and history were intuitive and quickly understood.

Areas for Improvement:

  • The icons for complex sentence translation and language settings were somewhat confusing and easily mixed up by users.
  • Font colors were difficult to read due to insufficient contrast.
  • Quiz cards flipped too quickly, making them hard to read.

Prototype Improvements Based on UX Evaluation

  • Enhanced readability by changing hint and instructional text to darker colors for greater contrast.
  • Implemented a bulk-delete feature in the History section for easier user operation.
  • Redesigned icons for translation and language-switching functions to improve clarity.
  • Added a hover-to-translate feature, addressing user suggestions for interactive translations.
  • Extended the display time for quiz answers to ensure sufficient reading time.

UX Evaluation Tasks

Participants performed the following tasks:

  • Activated the plugin from the main interface.
  • Changed the default system language.
  • Interacted with and explained their understanding of various plugin functionalities, including quizzes and translations.

UX Evaluation Process

We conducted our UX evaluation using the Co-Discovery method. Two participants simultaneously explored our prototype while verbalizing their thought processes. This method allowed us to capture detailed insights into usability concerns and discover issues that might not be immediately obvious.

UX Evaluation Quotes and Metrics

Anonymized User Quotes:

"Maybe display both languages side by side? Right now, it's only in English."
"It was a bit unclear at first what this option does—is it for switching between English and Chinese?"
"Quiz settings are intuitive, but I think translation could be more straightforward."

Informal UX Metrics:

  • Quiz accuracy: 100% (Both users correctly interacted with quiz features.)
  • Average time spent identifying the default language option:
    • User 1: 96 seconds
    • User 2: 73 seconds
  • Time spent exploring settings:
    • User 1: 181 seconds
    • User 2: 232 seconds

Future Steps

  • Interactive Inline Translation: Users expressed interest in a text-highlight translation feature, which would enhance real-time interaction beyond the current capabilities.
  • Improved Multilingual Support: Clearer language labels and multilingual interface options (e.g., Japanese-English or Chinese-English) were requested, requiring further development in backend localization.
  • Adaptive Learning Analytics: Users showed interest in adaptive learning features, which would require backend integration and analytics.
  • Cross-Device Accessibility: Users suggested extending the plugin to mobile and tablet devices for broader accessibility.

Improved Prototype

Improved visual clarity and accessibility for activating the WebLinguo plugin.

Improved translation functionality with clearer labeling and intuitive usage.

Enhanced history section demonstrating bulk delete feature.

Enhanced settings interface with clearer options for language and quiz customization.

Quiz timing extended to allow users sufficient time for comprehension and learning reinforcement.

Newly added highlight-to-translate feature, allowing immediate translations by highlighting text.

Future Steps

  • Inline text-highlight translations: Requested by users for convenience, this requires backend and frontend development to achieve real-time translation interaction.
  • Expanded multilingual support: Clearer labeling and additional language pairings will improve usability for diverse user groups.
  • Mobile compatibility: User interest indicated the importance of developing a responsive, cross-device compatible plugin to extend accessibility beyond desktop browsers.
